# TriAgent Role Policy

## 강제 규칙

1. 역할 토론이 끝나면 모델이 선택지에서 역할을 자동 확정하며, 사용자는 대시보드에서 언제든 다시 지정할 수 있다.
2. 활성 모델끼리는 같은 역할을 동시에 맡을 수 없다.
3. 역할 지정은 선택 사항이다. 미지정 모델은 일반 작업자로 정상 실행하며 역할 범위 강제를 적용하지 않는다.
4. 코드 리뷰어와 통합 조정자는 읽기 전용으로 동작한다.
5. 사용자가 역할을 지정한 모델이 담당 범위를 벗어난 파일을 변경하면 해당 턴을 실패 처리하고 세션을 일시정지한다.
6. 담당 밖 작업은 `CALL_AGENT`로 적절한 역할의 모델에 넘긴다. 담당 모델이 없으면 사용자에게 역할 변경을 요청한다.
7. 자동·수동 역할 변경은 실행 중에는 허용하지 않으며 세션의 `Roles.md`에 현재 배정과 이력을 기록한다.

## 저장 구조

- 이 파일은 모든 세션이 공유하는 전역 정책이다.
- 각 세션은 `roundtable_memory/<session-id>/Roles.md` 하나에 현재 역할과 변경 이력을 저장한다.
- 역할 선택지는 코드의 `ROLE_CATALOG`에서 관리한다. 역할마다 별도 MD를 만들지 않는다.
